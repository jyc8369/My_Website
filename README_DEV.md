## 1. 프로젝트 개요

### 프로젝트명

jyc8369 Personal Project Showcase

### 프로젝트 유형

정적 웹사이트

### 프로젝트 목적

개인 프로젝트를 효과적으로 전시하기 위한 인터랙티브 쇼케이스 웹사이트를 제작한다.

일반적인 개발자 포트폴리오처럼 경력, 기술 스택, 자기소개를 중심으로 구성하지 않는다.

프로젝트 자체를 하나의 제품처럼 소개하며, 방문자에게 강한 시각적 경험을 제공하는 것을 목표로 한다.

### 금지

* 불필요한 3D 효과

---

## 애니메이션 특징

애니메이션은 매우 절제된 방식으로 사용합니다.

주요 속성은 아래 3가지를 중심으로 구성합니다.

* opacity
* transform
* scale

대부분의 애니메이션 시간은 아래 범위를 기본값으로 사용합니다.

* transition: 200ms~300ms

과도한 패럴랙스나 3D 효과는 거의 사용하지 않습니다.

# 최종 정의

이 프로젝트는 개발자 포트폴리오가 아니다.

다양한 프로젝트를 제품 소개 페이지처럼 전시하는 인터랙티브 쇼케이스 웹사이트이며, 방문자에게 "디자인이 좋다"와 "재미있는 프로젝트를 만든다"라는 인상을 남기는 것을 최우선 목표로 한다.

# 개발 방향성 (GitHub Pages 정적 호스팅 기준)

이 프로젝트는 **GitHub Pages**로 배포하므로, 서버 실행이 필요한 기능 없이 동작하는 **정적 사이트**로 개발합니다.

## 핵심 원칙

1. 서버 사이드 코드 금지
- Node/Express, Python/Flask, PHP, DB 연결 등 서버 의존 로직은 사용하지 않습니다.

2. 클라이언트만으로 동작
- HTML/CSS/JavaScript만으로 화면 렌더링과 사용자 상호작용을 구현합니다.

3. 경로는 상대 경로 우선
- GitHub Pages의 저장소 하위 경로에서 깨지지 않도록 리소스 링크는 상대 경로를 우선 사용합니다.

4. 빌드 산출물은 정적 파일
- 배포 결과물은 `main.html` 및 정적 자원(`.css`, `.js`, 이미지 등)으로만 구성합니다.

5. 환경 변수/비밀키 사용 금지
- 프런트 코드에 토큰/키를 넣지 않습니다. 공개 저장소 기준으로 안전한 구조를 유지합니다.

## 구현 가이드

- 라우팅은 서버 라우팅 대신 앵커(`#`) 기반 또는 단일 페이지 구조를 고려합니다.
- 데이터는 정적 JSON 파일 또는 외부 공개 API(인증 불필요)만 사용합니다.
- 폼 전송이 필요하면 백엔드 직접 구현 대신 외부 폼 서비스 연동을 검토합니다.
- 새 기능 추가 시 "정적 호스팅에서 동작 가능한가"를 먼저 확인한 뒤 코드 작성합니다.

## PR/리뷰 체크리스트

- [ ] 로컬 서버 없이 정적 파일만으로 동작하는가?
- [ ] 서버 의존 코드(백엔드/DB/SSR)가 포함되지 않았는가?
- [ ] 경로가 GitHub Pages 배포 경로에서 깨지지 않는가?
- [ ] 민감 정보(키/토큰/비밀값)가 코드에 포함되지 않았는가?
